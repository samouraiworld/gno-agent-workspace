**TL;DR:** A new ~1090-line operator guide (`docs/validators/tmkms-quickstart.md`) that walks you from an empty machine to a gnoland validator whose consensus key is held by tmkms instead of sitting on the node, plus a 6-line cross-link from the existing `tmkms.md` reference. It is procedural and security-first: one locked-down `gnoland` system user, `0600` secrets, a transport choice (same-host Unix socket vs firewalled TCP), and three signer backends (YubiHSM recommended, softsign for labs, Ledger for advanced).

**Verdict: REQUEST CHANGES** — the softsign lab (Part C) tells you to run `gnoland start -lazy`, but master rejects `-lazy` whenever the tmkms listener is enabled, so that path fails at its final step; and several UDS sections state gnoland "requires" a non-empty `allowed_kms_pubkeys`, which is false on `unix://` (it is ignored there, with a warning). Everything else — every flag, default, config key, and the example pubkey→address pair — verifies against master.

## Summary
This supersedes #5797 (auto-closed when its base branch `feat/tmkms-compat/03-listener-integration` was deleted after the tmkms listener work merged in #5718). It is the same guide rebased onto master, but reworked across 12 commits into a single-`gnoland`-user model with a `/opt/gno` clone for binaries+GNOROOT. The listener-integration PR (#5718) that merged in the meantime added a guard (`errLazyTmkmsListener`) the original review never saw: `gnoland start -lazy` now refuses to derive a genesis when `tmkms_listener` is enabled. Part C's softsign flow enables that listener and relies on `-lazy` to seed genesis, so it stops at the last command. Separately, the empty-allowlist rule on master is scheme-dependent (required on `tcp://`, ignored on `unix://`), which the UDS sections get backwards in their stated reason.

## Glossary
- **privval / tmkms_listener** — gnoland's listen-mode privval: gnoland listens, the external signer (tmkms) dials in and signs votes/proposals.
- **consensus key** — the validator's signing key; in tmkms mode it lives in the signer (HSM/Ledger/softsign file), and only its pubkey goes into genesis.
- **`-lazy`** — `gnoland start` flag that auto-generates missing node files and derives a single-validator genesis from the locally available signing key.
- **double-sign / HRS gate** — the tmkms state file tracks the last height/round/step signed; rolling it back can cause a slashable double-sign.

Carried from the #5797 review (code moved from `config.go` to `config.go:221`); still applies. Confirmed behaviorally: the chmod is wrapped in an error branch that only warns.
