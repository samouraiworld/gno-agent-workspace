## Glossary

- `SignerClient` — implements `types.PrivValidator` over the privval socket; caches consensus pubkey at `Init()`, re-verifies on every reconnect via a connection-generation counter.
- `SignerListenerEndpoint` — listens for tmkms to dial; owns the held conn, deadline management, ping keepalive, and accept/reconnect machinery.
- `instanceMtx` vs `connMtx` — `instanceMtx` serializes public method access (SendRequest, Init, SignVote); `connMtx` guards the conn pointer alone. Both held in tandem during sign sequences.
- `connGen` — atomic uint64 on `signerEndpoint`; bumped on each new conn install. `verifiedGen` (on `SignerClient`) tracks the last gen at which the signer's pubkey was verified.
- `HRSGuardedSigner` — gnokms-side wrapper enforcing strict (height, round, step) monotonicity persisted on the signer host; mirrors tmkms's `consensus.json` gate. Not wired into gnokms's `RunSignerServer` flow in this PR's scope but the `flags.go` `state-file` flag and `server.go` `NewHRSGuardedSigner(...)` wrap are present.

## Fix

Before this PR, gno.land validators had only two privval options — local file or gnokms (alpha-tier, fail-open allowlist default, dial-direction inverted, HRS authority on the validator host). After, operators add a `[priv_validator.tmkms_listener]` config block; `NewPrivValidatorFromConfig` branches into [`newTmkmsListenerPrivValidator`](https://github.com/gnolang/gno/blob/9b3af52b3/tm2/pkg/bft/privval/config.go#L178-L234) · [↗](../../../../../.worktrees/gno-review-5625/tm2/pkg/bft/privval/config.go#L178-L234) which wires `TCPListener` → `SignerListenerEndpoint` → `SignerClient` → `RetrySignerClient` and returns the chain a `types.PrivValidator`. The load-bearing constraint: in tmkms mode the validator does NOT carry a local FileState gate (tmkms's `consensus.json` owns HRS authority — a redundant validator-side gate would be a misconfiguration footgun per the inline rationale at [`config.go:157-159`](https://github.com/gnolang/gno/blob/9b3af52b3/tm2/pkg/bft/privval/config.go#L157-L159) · [↗](../../../../../.worktrees/gno-review-5625/tm2/pkg/bft/privval/config.go#L157-L159)). Comment the invariant: "pingLoop blocks on instanceMtx; concurrent sign requests are serialized, not raced".

## Suggestions

- **[`HRSGuardedSigner` ergonomics — expose `LastHRS()` accessor]** The `state.String()` log line at server.go:128 reaches into the private field; a `LastHRS() (height int64, round int, step state.Step)` method would let operator dashboards / health checks query the gate's current position without crossing the abstraction. Useful for operations even if not strictly required.

- **[Consider an optional "read-only HRS observer" on the validator side]** Even in tmkms mode, having the validator persist its OWN observation of (height, round, step) per sign call — without using it as a gate — would let operators compare tmkms's `consensus.json` to the validator's view and catch divergence early. The reasoning at [`config.go:157-159`](https://github.com/gnolang/gno/blob/9b3af52b3/tm2/pkg/bft/privval/config.go#L157-L159) · [↗](../../../../../.worktrees/gno-review-5625/tm2/pkg/bft/privval/config.go#L157-L159) against a gate is sound; an observation log is not a gate. Defense-in-depth without the footgun. Optional, opt-in flag.

- **[`tmkms.toml` snippet in docs — pin `reconnect = true`]** The example at [`tmkms.md:119-125`](https://github.com/gnolang/gno/blob/9b3af52b3/docs/validators/tmkms.md#L119-L125) · [↗](../../../../../.worktrees/gno-review-5625/docs/validators/tmkms.md#L119-L125) sets `reconnect = true` but the integration test at [`tmkms_integration_test.go:106`](https://github.com/gnolang/gno/blob/9b3af52b3/tm2/pkg/bft/privval/upstream/tmkms_integration_test.go#L106) · [↗](../../../../../.worktrees/gno-review-5625/tm2/pkg/bft/privval/upstream/tmkms_integration_test.go#L106) uses `reconnect = false`. - **[ADR-003 already exists — link it from `docs/validators/tmkms.md`]** Operators reading the deployment guide would benefit from a "for the design rationale, see ADR-003" pointer; currently the doc doesn't reference the ADR.
