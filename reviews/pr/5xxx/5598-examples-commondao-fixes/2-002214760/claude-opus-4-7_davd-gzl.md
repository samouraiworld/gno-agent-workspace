## Summary

The two newest commits change `ExecFunc` from `func(realm) error` to `func(pkgPath string) error` (`51948130b`) and add `realm.IsCurrent` gating on `Execute` (`8243bc930`), but the sibling realm under `r/nt/commondao/v0` and a filetest were not migrated and now fail type-check — CI's `gno-checks / test` and `gno2go` are red because of this PR, not flakes. Stale README (`README.md:131`) and live realm code (`proposal_members.gno:122`, `proposal_subdao.gno:99,163`, `render.gno:344,389`, `filetests/z_6_a_filetest.gno:32`) all need updating before merge.

## Glossary

- **ExecFunc** — `func(pkgPath string) error`; the function returned by `Executable.Executor()`. Used to live as `func(realm) error` and was invoked via `cross(rlm)`; now invoked with `rlm.PkgPath()` so the executor runs in its own declaring realm context.
- **Sealable** — `Seal() Sealable; IsSealed() bool`. Returns an immutable copy; further writes panic. Replaces the round-1 `Readonly*` wrappers.
- **canonical type** — concrete type defined inside `p/nt/commondao/v0` (e.g. `*memberStorage`). `New(...)` panics if a non-canonical interface value is supplied; `NewUnsafe(...)` skips the check.
- **member grouping** — second-tier storage attached to a `MemberStorage`. As of commit `e7ed3a8d`, ungrouped `Size()`/`Has()` no longer reach into groups — see Warnings below.

For realms that intentionally allow multi-group membership (`singleUserRole == false` in `boards2/permissions`, the default), every tally using `GetTotalMemberStorageSize` for the quorum denominator is biased toward rejection. Fix: either ship the unique iterator now (a small AVL of `address → struct{}` walked across groups suffices) and add a `GetUniqueMemberStorageSize`, or rename to `GetMemberStorageSlotCount` so the bias is obvious from the symbol name. As long as the canonical `TallyByAbsoluteMajority` in `definition.gno` calls this, the bias is the default behaviour.
  </details>
