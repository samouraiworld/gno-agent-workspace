## Summary

The harness is a separate Go module: each `expected/<slice>.yaml` names a rule and a vulnerable/fixed fixture pair, and `internal/auditpattern` runs `gno test` on each fixture plus a line-based pattern rule, asserting the vulnerable side flags and the fixed side stays clean. The rules are deliberately heuristic (text over a gofmt-normalized, string/comment-blanked view of the source). The reference realm and the security-guide additions ([§5.8](https://github.com/gnolang/gno/blob/e8281bcbe/docs/resources/gno-security-guide.md?plain=1#L330) `OriginCaller`-as-auth, §5.9 raw text in `Render`) are the builder-facing payload; the storage docs reframe "prefer AVL" into "choose storage by access pattern." `AGENTS.md` adds a Security trigger table mapping seven vulnerability families to "what to look for," plus a pointer to run the harness against unfamiliar realm code. The PR is fully additive and touches no consensus, VM, or stdlib runtime code, so blast radius is confined to docs, examples, and a dev tool. The two remaining findings are integrity gaps in the harness-as-contract claim, not runtime risk.

## Glossary

- crossing function — a `.gno` function whose first parameter is `cur realm`; entered via `f(cross, ...)`, which mints a fresh `cur` and shifts `PreviousRealm`.
- `cur.IsCurrent()` — true only when `cur` is the topmost crossing frame's live realm token; at a properly entered crossing function it is true by construction.
- `cur.Previous()` — the realm that crossed into the current frame; reads the captured prev field verbatim, with no IsCurrent validation of its own.
- `IsUserCall()` — pure on the stored `pkgPath`: true only for a direct EOA `maketx call`, false for code realms and ephemeral `maketx run` realms.
- `OriginCaller()` — the transaction signer; same across intermediate realms, so unsafe as an immediate-caller identity. Lives in `chain/runtime/unsafe`.

- [`run.go:146-391`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L146-L391) · [↗](../../../../../.worktrees/gno-review-5835/misc/audit-pattern-harness/internal/auditpattern/run.go#L146) — deep pass over the eight rules surfaced more undocumented false positives and negatives, each verified with `RunRule` on gofmt-clean input.
  <details><summary>details</summary>

  False negatives, the dangerous direction for a security aid:
  - **Closure-wrapped guard.** [`currentGuardHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L173) and [`paymentUserCallHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L252) set the function-wide `seen` flag on any `.IsCurrent()` / `.IsUserCall()` token, including one inside a nested function literal that never runs, so an unguarded `.Previous()` / `OriginSend()` in the outer body is missed. Distinct from the README's helper-function caveat: here the guard is in the same function, in dead closure code.
  - **Multi-line signature.** The block scanners [`renderMarkdownEscapeHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L188), [`interfaceRealmParamHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L285), and [`renderMapIterationHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L338) open the block on the keyword line and close it at once when `braceDepth <= 0`, so a `func Render(` whose `{` is on a later line returns the raw path unflagged.
  - **Multi-value return.** [`exportedPointerLeakHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L330) requires the literal `) *`, so `func GetRegistry() (*Reg, error)` and named-return `func GetT() (out *T)` are missed; only single-return `) *T` is caught.

  False positives:
  - **`callback_param` flags a function that returns a func.** [`callbackParamHits`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L280) matches `func ` plus ` func(`, so `func Make() func()` is flagged though it takes no callback, and inconsistently `func Make() (func(), error)` is not.
  - **`render_map_iteration` count inflation.** [The inner loop](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L378) appends one hit per map name that is a substring of `range <name>`, so a single `range scoreboard` loop with maps `score` and `scoreboard` yields two hits, which also breaks the `len(hits) == want_pattern_hits` check.
  - **Scans `_test.gno`.** [`gnoFiles`](https://github.com/gnolang/gno/blob/e8281bcbe/misc/audit-pattern-harness/internal/auditpattern/run.go#L482) walks every `.gno` including test files, so a pattern in test code is flagged when auditing a real realm.

  All polish on a disclaimed heuristic tool. The README's Known limitations cover only `current_guard`, `payment_user_call`, and `render_markdown_escape`; tighten the heuristics or extend the caveats to the rest.
  </details>
