**TL;DR:** Lets a realm mint an identity token for one of its internal actors, say a DAO inside a registry, so downstream callees can tell that actor apart from the host realm and give it its own chain address. Separately fixes a hole where a called realm could build a coin-spending banker over whoever called it and drain them.

**Verdict: REQUEST CHANGES** — the `NewBanker` sub-token gate answers "is this a sub-token" with an interpreted call instead of the native accessor this PR adds, costing every `OriginSend` and `RealmIssue` banker construction ~295k gas, 15% of a minimal payment transaction, for a one-line swap; plus a derivation helper whose own comment promises a funds-safety invariant it does not hold, and a drift-prevention test that silently stops checking on the exact edit it exists to catch. `Sub` has not shipped in a release (neither v1.0.0 nor v1.1.0 contains it, and neither is an ancestor of master), so a follow-up lands alongside it in the same upgrade rather than needing a second one.

## Summary
Adds `cur.Sub(subpath)` to the `realm` interface: it returns a first-class realm value whose pkgpath is the synthesized `host#subpath` and whose address is `chain.PackageAddress(host#subpath)`. `#` is reserved (rejected at package validation), so a synthesized path can never collide with a deployed package and exact-match pkgpath auth is never silently broadened. The sub-token's `Previous()` equals the host's `Previous()`; an internal parent reference anchors `IsCurrent`/`cross` staleness. Independently, `NewBanker` now requires `rlm.IsCurrent()`: without it a callee could pass `cur.Previous()` (address = caller) into a `RealmSend` banker and drain its caller through the `pkgAddr==from` gate. Also removes the legacy `cross1` migration sentinel. Consensus-affecting: the `gno0p9` realm interface grows two methods (additive on a sealed interface), `cross1` stops type-checking, and — unremarked — the banker's new sub-token gate raises the floor price of the two banker types that carry payments.

## Examples
| Call | Result |
|---|---|
| `cur.Sub("dao/42").PkgPath()` | `gno.land/r/nt/commondao/v0#dao/42` |
| `cur.Sub("dao/42").Address()` | `chain.PackageAddress("gno.land/r/nt/commondao/v0#dao/42")` |
| `cur.Sub("dao/42").Previous()` | `cur.Previous()` (host not a chain step) |
| `cur.Sub("Dao")` / `"a b"` / `"a#b"` / `".."` / `"dao/"` | panic (frozen grammar) |
| `cur.Sub("dao/42")` on an `/e/…/run` cur | panic (ephemeral cannot mint) |
| `chain.DerivePkgSubAddr("gno.land/e/g1…/run", "dao")` | **accepted** — derives an address nothing can ever mint |
| `sub.Sub("b")` (sub-of-sub) | panic (receiver pkgpath already synthesized) |

## Glossary
- **cur realm** — the interface handle a crossing function receives, carrying the live realm-context identity.
- **IsCurrent** — true iff the realm value is the topmost live crossing frame's cur; the anti-spoof gate `cross(rlm)` also uses.
- **sub-realm token** — a realm value minted by `cur.Sub(subpath)` with a synthesized `host#subpath` pkgpath and a hash-derived address.
- **heap item** — the value backing a realm handle; `Sub`'s entry guard compares it by pointer.
- **ephemeral realm** — the short-lived `/e/<addr>/run` realm a `maketx run` executes under.

## Fix
Two independent changes. `NewBanker` gains an `rlm.IsCurrent()` gate at [`banker.gno:101`](https://github.com/gnolang/gno/blob/b940037d1/gnovm/stdlibs/chain/banker/banker.gno#L101) · [↗](../../../../../.worktrees/gno-review-5890/gnovm/stdlibs/chain/banker/banker.gno#L101) plus a sub-token banker-type restriction at [`banker.gno:111-115`](https://github.com/gnolang/gno/blob/b940037d1/gnovm/stdlibs/chain/banker/banker.gno#L111-L115) · [↗](../../../../../.worktrees/gno-review-5890/gnovm/stdlibs/chain/banker/banker.gno#L111). `Sub` is a new `.grealm` native at [`uverse.go:1712`](https://github.com/gnolang/gno/blob/b940037d1/gnovm/pkg/gnolang/uverse.go#L1712) · [↗](../../../../../.worktrees/gno-review-5890/gnovm/pkg/gnolang/uverse.go#L1712) with a strict entry guard, backed by the reserved-`#` check in [`mempackage.go:1140`](https://github.com/gnolang/gno/blob/b940037d1/gnovm/pkg/gnolang/mempackage.go#L1140) · [↗](../../../../../.worktrees/gno-review-5890/gnovm/pkg/gnolang/mempackage.go#L1140). The load-bearing constraint is that the subpath grammar is frozen at introduction: loosening later is additive, tightening would strand funds already sent to a now-illegal sub-address.

## Benchmarks / Numbers
`NewBanker(BankerTypeOriginSend, cur)` measured end-to-end through `gnokey maketx call`, one realm, two entry points, shipped gate vs the gate on `rlm.Subpath() != ""`:

| Transaction shape | Shipped | `Subpath()` | Delta |
|---|---|---|---|
| `NewBanker(OriginSend)` alone | 1,958,206 | 1,663,192 | −295,014 |
| same, tx already used `strings` | 1,672,469 | 1,561,969 | −110,500 |

The cost splits in two. ~185k is the one-time load of the interpreted `strings` package that the [`chain.SplitPkgSubPath`](https://github.com/gnolang/gno/blob/b940037d1/gnovm/stdlibs/chain/address.gno#L37) · [↗](../../../../../.worktrees/gno-review-5890/gnovm/stdlibs/chain/address.gno#L37) call forces at `NewBanker` time, and it amortizes away if the transaction touches `strings` anywhere else. The remaining ~110k is the interpreted call itself and never amortizes. For scale, a no-op crossing call in the same realm is ~980k, so the surcharge is ~15% of a minimal payment transaction even in the amortized-best case.
