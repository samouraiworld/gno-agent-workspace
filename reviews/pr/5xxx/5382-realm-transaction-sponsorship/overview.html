Realm transaction sponsorship: PayGas + PayStorage

Plain-language overview of gnolang/gno#5382 · generated by Claude Opus 4.8 · explainer only, no review verdict

Today every gno.land transaction needs the signer to already hold gnot to pay for gas. This PR lets a realm pay a user's gas and storage out of its own balance, so a user with zero gnot can still transact. The realm decides while the transaction runs whether to sponsor, so it can charge in another token, check a whitelist, or rate-limit first, then commit to pay.

The idea in one picture

  NORMAL TX (today)                         SPONSORED 0-FEE TX (this PR)
  ─────────────────                         ───────────────────────────
  user needs gnot                           user needs 0 gnot
       │                                         │  gas-fee = 0
       │ pays gas-fee up front (ante)            │  (validator must opt in)
       ▼                                         ▼
  ┌───────────┐                            ┌───────────────────────────┐
  │  run msgs │                            │ run on a CREDIT window     │
  └───────────┘                            │ (MaxGasCreditPerTx gas)    │
       │                                   │   realm runs its logic …   │
       ▼                                   │   realm calls PayGas(max)  │◄─ realm commits to pay
  fee already taken                        │   realm calls PayStorage() │   (optional)
                                           └───────────────────────────┘
                                                 │ tx succeeds?
                                    ┌────────────┴────────────┐
                                 yes│                       no│
                                    ▼                         ▼
                           settle: realm pays        revert everything,
                           actual gas + storage       realm pays NOTHING
                           (capped at its maxFee)      (free run bounded
                           user pays 0                 by the credit window)

If a 0-fee transaction never calls PayGas, it has no payer, so it is rejected, at both mempool-admission time (CheckTx) and block time (DeliverTx). A block proposer cannot force-include a free run that skips PayGas.

Concepts

runtime.PayGas(maxFee)
A native a realm calls mid-execution to say "I'll pay this tx's gas, up to maxFee ugnot." It shrinks the gas meter to what maxFee buys and records the realm as payer. No-op in a normal fee-paying tx.
runtime.PayStorage(maxDeposit)
Same idea for storage deposits (the gnot locked when a realm grows on-chain state), capped at maxDeposit. Independent of PayGas: either, both, or neither, and the two may even be paid by different realms.
Credit window (Block.MaxGasCreditPerTx)
A consensus parameter: how much gas a 0-fee tx may burn before a realm commits to pay. Bounds the free work validators absorb for a tx that turns out to have no payer. Default 0 = feature off. Capped at Block.MaxGas so one sponsored tx can't be bigger than a whole block.
Two gates
The feature needs both the consensus param MaxGasCreditPerTx > 0 (chain-wide, deterministic) and a per-validator AllowZeroFeeTxs opt-in that only decides whether that validator admits 0-fee txs to its mempool. Block validation does not depend on the opt-in, so validators stay in consensus.
Fee.SponsorStorage
A tx flag that defers every message's storage deposit to end-of-tx, so one PayStorage can cover storage from all messages in a multi-message tx. Off by default (per-message behaviour, backward compatible). Restricted to single-signer txs.
Settlement
After a successful tx, the realm is charged ceil(gasUsed × gasPrice) capped at maxFee, routed to the same fee collector normal fees use. On failure everything reverts and the realm pays nothing, which stops an attacker from engineering a failure to drain the realm.

From the user's side

Before — needs gnot first
1. Acquire gnot (faucet, exchange, transfer)
2. Sign tx with a non-zero gas-fee
3. Fee deducted from the user's wallet
Gasless only via off-chain co-signers.
After — realm sponsors
1. User holds 0 gnot (account must already exist)
2. Sign tx with gas-fee = 0
3. Realm runs its logic, calls PayGas, pays
No off-chain infra; realm decides on-chain.
