A from-scratch multi-lens pass (red-team / blue-team / correctness / consensus-persisted-state) plus an independent verification of the consensus-safety mechanics the incremental rounds asserted. It confirms the round-4 APPROVE and adds one new finding the four earlier rounds missed: a Nit-level, order-dependent Go-parity gap in unexported-method *selection* (`FindEmbeddedFieldType`), safe-direction and near-unreachable.

**TL;DR:** When you embed a type by its name inside `struct{...}`, that name becomes the field's name; for `interface{...}`, embedding contributes only the embedded type's methods. This PR makes struct embeds keep the name exactly as written (so a type alias is its own field, distinct from the aliased type) and makes interface embeds collapse to their flattened method set (so `interface{ Stringer }` equals `interface{ Str() string }`), both matching the Go compiler. A per-method origin-package field keeps an unexported method from one package distinct from a same-named one elsewhere, so a type can't satisfy or select another package's sealed interface.

**Verdict: APPROVE** — correct, consensus-safe, and well-tested; the one deep-mode finding is a false-reject-only, degenerate-case selection asymmetry in `FindEmbeddedFieldType` that neither bypasses a sealed interface nor affects consensus, with a tested localized fix. Independent checks confirm the wire format stays byte-identical for every FieldType that doesn't carry a cross-package unexported method, and the flattened TypeID is deterministic.

## Summary
Round 5 re-derives the change's safety from scratch rather than trusting the incremental rounds. The consensus surface holds: `FieldType.MarshalBinary2` guards the new field with [`if goo.PkgPath != ""`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/pb3_gen.go#L14255) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/pb3_gen.go#L14255), so every FieldType that leaves `PkgPath` empty — all struct fields, exported methods, same-package methods — encodes byte-for-byte as before; only cross-package-hoisted unexported methods (a shape that could not exist pre-flatten) carry it. Identity is likewise preserved for those: [`idName`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L426) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L426) routes through [`originPkg`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L414) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L414), which falls back to the enclosing package when `PkgPath` is empty, i.e. the exact pre-PR TypeID. The flattened identity is deterministic: flatten emits methods in input-slice order (the `seen` map is dedup-only), and [`sortForPackage`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L439) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L439) keys the sort identically to emission and panics on a duplicate qualified name, so there is no map-iteration dependence.

The one gap is in *selection*, not identity. [`FindEmbeddedFieldType`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L1062-L1077) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L1062) scans the flattened method list and binds a name to the first entry that matches, gating only that one on origin package. When an interface holds two same-named unexported methods from different packages, selection succeeds or fails by source order. Satisfaction ([`VerifyImplementedBy`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L1096) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L1096)) loops all methods with AND-semantics, so it is order-independent and the sealed guarantee is unaffected.

## Glossary
- TypeID: a type's canonical string identity; type equality and persisted on-chain state both key off it. For interfaces it folds in each method entry.
- flatten (interface): replacing an embedded-interface entry with its individual methods, so identity is the method set, not the embed name.
- sealed interface: an interface containing an unexported method, satisfiable only from the method's defining package.
- stamp: setting `FieldType.PkgPath` on a method to record its origin package, used when the method has been hoisted out of another package's interface.
- Amino: gno's deterministic serialization codec; the on-chain state hash is over its binary encoding, not its JSON.
- filetest: a VM-run `.gno` file asserted against `// Output:` / `// Error:` golden directives.

## Nits
- **[minor: valid Go rejected depending on method order]** [`types.go:1062-1077`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/types.go#L1062-L1077) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/types.go#L1062) — an interface that embeds a foreign sealed interface and also declares its own same-named unexported method selects that method by source order; Go binds to the caller's own method either way.
  <details><summary>details</summary>

  `FindEmbeddedFieldType` stops at the first flattened entry whose name matches `n` and applies the origin-package gate only to that entry. For `interface{ ifaceext.Sec; sec() int }` (foreign `ifaceext.sec` before main's own `sec`), the first `sec` is the inaccessible foreign one, so `x.sec()` from `main` is rejected with `cannot access interface {sec func() int; sec func() int}.sec from main`; the own-first spelling binds to `main.sec` and compiles. Go accepts both orders, resolving `x.sec()` to the caller-package method regardless of position (confirmed with a go/build oracle). The direction is safe — it over-rejects rather than granting access, and satisfaction is unaffected — and the trigger is degenerate: the interface is unsatisfiable by any concrete type, since no type can carry both a `main`-unexported and an `ifaceext`-unexported `sec`. A localized fix (continue scanning past an inaccessible unexported match for an accessible same-named one; return the access error only if every match was inaccessible) turns the case green and keeps `iface_embed_xpkg_access` / `iface_embed_xpkg` / `iface_embed_xpkg2` green. Test: [`iface_embed_sel_order.gno`](https://github.com/samouraiworld/gno-agent-workspace/blob/main/reviews/pr/5xxx/5739-preserve-embedded-alias-name/5-a00dde6b3/tests/iface_embed_sel_order.gno) · [↗](tests/iface_embed_sel_order.gno) (red at `a00dde6b3`, green under the fix; [repro](comment_claude-opus-4-8.md)); Go oracle [`iface_embed_sel_order_go.go`](https://github.com/samouraiworld/gno-agent-workspace/blob/main/reviews/pr/5xxx/5739-preserve-embedded-alias-name/5-a00dde6b3/tests/iface_embed_sel_order_go.go) · [↗](tests/iface_embed_sel_order_go.go). Fix: for an unexported selector, prefer the flattened entry whose origin package equals the caller before reporting an access error.

## Verified
- **Wire format is byte-identical for the common case.** `FieldType.MarshalBinary2` and `SizeBinary2` both guard on `goo.PkgPath != ""` ([pb3_gen.go:14255](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/pb3_gen.go#L14255), [:14338](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/pb3_gen.go#L14338) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/pb3_gen.go#L14255)), so every struct field, exported method, and same-package method serializes exactly as before; the amino binary encoding is the consensus surface. The `"PkgPath": ""` seen in the five `zrealm_*` goldens is the JSON representation only, which is display/RPC, not the state hash — each golden change is the mechanical consequence of the new field and is justified.
- **No stray TypeID drift.** `idName` → `originPkg` falls back to the enclosing package for an empty stamp, reproducing the pre-PR identity; only the intended interface-flattening changes TypeIDs, the acknowledged `fix(gnovm)!` break. The gate that keeps a decoded pre-flattening interface from emitting a split identity sits ungated at the decode boundary [`fillType`](https://github.com/gnolang/gno/blob/a00dde6b3/gnovm/pkg/gnolang/realm.go#L1827) · [↗](../../../../../.worktrees/gno-review-5739/gnovm/pkg/gnolang/realm.go#L1827), reached by every stored type.
- **Determinism holds.** Flatten output order is the input slice order (the `seen` map is a dedup lookup, never drives order); `sortForPackage` keys the sort on the same `idName` the TypeID emits and panics on a duplicate qualified name, so identity does not depend on Go map iteration or construction route.
- **Gas unchanged.** `OpCPUSlopeStructType` (30) and `OpCPUSlopeInterfaceType` (27) and the `incrCPU` calls are byte-identical to master; the charge at the top of `doOpInterfaceType` fires before the flatten work. The PR touches no gas path.
- **The selection asymmetry is real and safe.** The shipped filetest is red at `a00dde6b3` (`cannot access ...sec from main`); the localized fix makes it green without regressing the PR's own cross-package selection tests. `VerifyImplementedBy` iterates all methods, so satisfaction is order-independent and no type can bypass a sealed interface through ordering.
- Green at `a00dde6b3`: `TestFlattenInterfaceMethods`, `TestInterfaceType_UnflattenedIsHardError`, `TestDoOpInterfaceType_Flattens`, `TestDoOpStructType_EmbedNames`, `TestInterfaceTypeID_PkgPathProvenance`, and the filetests `iface_embed_conflict`, `iface_embed_id`, `iface_embed_more`, `iface_embed_provenance`, `iface_embed_xpkg`, `iface_embed_xpkg2`, `iface_embed_xpkg_access`, `alias2`, `alias4`, `alias5`, `alias6`, `alias7`.
