# Review History

Answer questions about work already reviewed, from `reviews/` on disk. Never re-review to answer a history question, and never fetch from GitHub for anything the corpus already holds.

**Input:** `$ARGUMENTS` — a PR number, a free-text search term, or one of the mode words below. Empty means summarize recent activity.

## Corpus layout

- `reviews/pr/<thousand>xxx/<number>-<slug>/<round>-<short-sha>/` — gno PRs. `<round>` counts up per PR; the highest is current.
- `reviews/<repo>/<number>-<slug>/<round>-<sha>/` — non-gno repos (e.g. `reviews/onboarding-bot/`).
- Each round holds `review_<model>_<reviewer>.md` and `comment_<model>.md`. Older rounds may lack the `review_` prefix.
- `reviews/BATCH_STATUS.md` — the in-flight or last batch run. Transient; a concurrent session may own it.
- `reviews/README.md` and root `index.html` — generated by `scripts/build-indexes.sh`, never hand-edited.

Verdict lives on the `**Verdict: ...**` line of the review file. A posted draft carries a `Posted: <url>` line in its `comment_*.md`; absent that line, it was never posted.

## Modes

**PR lookup** (`$ARGUMENTS` is a number). Report: reviewed or not, every round with its sha, the current verdict, whether it was posted and where, and the headline findings from the latest round. Name the round directory so the user can open it.

```bash
ls -d reviews/pr/*/<number>-*/ 2>/dev/null || echo "not reviewed"
```

**Search** (free text). Grep findings across all reviews, report PR number, severity section, and the one-line TL;DR, most recent first. Use it for "did we ever flag X", "what did we say about gas metering", "which PRs touched the banker".

```bash
grep -rl '<term>' reviews/pr/*/*/*/review_*.md
```

**stats**. Verdict distribution, review count, how many were posted, which PR areas recur. Count from the latest round of each PR only, never every round, or re-reviews double-count.

**posted**. Only reviews actually sent to GitHub, with their URLs.

**recent [n]**. Last n reviews by round-directory mtime, with verdict and posted state.

## Listing one line per PR

Latest round, verdict, posted state. Adjust the glob to scope it.

```bash
for d in reviews/pr/*/*/; do
  n=$(basename "$d" | grep -oE '^[0-9]+')
  last=$(ls -d "$d"*/ 2>/dev/null | sort -V | tail -1); [ -z "$last" ] && continue
  rf=$(ls "$last"review_*.md 2>/dev/null | head -1); [ -z "$rf" ] && continue
  v=$(grep -m1 '^\*\*Verdict' "$rf" | sed 's/\*\*Verdict: //; s/\*\*.*//')
  p=$(grep -lq '^Posted:' "$last"comment_*.md 2>/dev/null && echo POSTED || echo draft)
  printf "%s\t%s\t%s\t%s\n" "$n" "$v" "$p" "$(basename $last)"
done
```

## Rules

- Read-only. This skill never writes a review, edits a draft, posts, or pushes.
- Cite `reviews/...` paths as clickable links so the user can open the source.
- A finding's severity comes from the section it sits under (Critical, Warnings, Nits, Missing Tests, Suggestions). Report the section, never re-rank.
- Verdicts are per-round. When rounds disagree, report the latest and say the earlier one was overturned, with both round numbers.
- The corpus reflects the reviewed sha, not current master. Before presenting a finding as live, check it still applies; say plainly when you have not checked.
- Don't infer a PR is unreviewed from a missing directory alone when the slug may differ; match on the leading number.
- Absent `Posted:` means not posted. Never guess from context.
- Findings recorded as Open questions were deliberately not posted. Say so rather than presenting them as raised with the author.
