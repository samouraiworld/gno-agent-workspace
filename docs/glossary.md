# Gno glossary

Shared vocabulary for gnolang/gno review and reporting, loaded by `skills/review.md`. A living doc: when a review needs a project-internal term, add a one-line entry here so the next review names it instead of re-explaining it. One terse line per term, alphabetical. Each term names a reusable concept the next review will meet again — a component, mechanism, or recurring pattern — not a function or symbol scoped to one PR; cite a specific symbol inside the definition when it pins the concept, never as the term itself. Prefer a plain, readable term over an opaque abbreviation (write "copy-on-write", not "COW"); keep an abbreviation as the term only when it is the codebase's own name (IAVL, ICS23, ABCI).

- addpkg: the transaction (`maketx addpkg`) that uploads a package or realm to the chain.
- Allocator: VM component tracking memory allocation and charging allocation gas; `fallbackAllocator` is the global, effectively-unbounded (`MaxInt64` budget) instance for pure-function or no-Machine paths.
- Amino: gno's deterministic serialization codec (`tm2/pkg/amino`) for on-chain object state, genesis, and RPC; unlike `encoding/json` it honors `,omitempty` on struct-typed fields, so stripping the tag changes JSON output.
- ante handler: the pre-execution stage (`tm2/pkg/sdk/auth/ante.go`) that runs for every tx before its messages: it verifies signatures, checks and increments the account sequence, enforces gas-wanted bounds and mempool fee minimums, and deducts the fee; its writes (notably the sequence increment) survive even a failed tx.
- app hash: the per-block commitment to application state (the multistore's Merkle root), agreed in consensus; two honest nodes computing different app hashes for the same block halts the chain.
- banker: stdlib API (package `chain/banker`, interface `banker.Banker`, constructed via `banker.NewBanker`) for issuing, sending, and burning coins from a realm.
- `baseStore` / `iavlStore`: the two tm2 backends of the VM's `defaultStore` (`gnovm/pkg/gnolang/store.go`), passed as `NewStore(alloc, baseStore, iavlStore)`. On chain `iavlStore` is the merkleized bptree mounted on `mainKey`, so its contents feed the app hash; `baseStore` is a `dbadapter` store mounted on `baseKey`, persisted and queryable but returning a nil commit hash, so its contents never do. Realm object bytes live in `baseStore` with only their escaped hashes in `iavlStore`; tooling collapses both roles onto one store (`NewStore(_, base, base)`).
- bigdec: the VM's arbitrary-precision decimal (`BigdecValue`, wrapping `apd.Decimal`) that backs an untyped float constant before it is typed; `ConvertUntypedBigdecTo` narrows it to `float32`/`float64` in constant space.
- block: a GnoVM scope frame (`Block`, `values.go`). A runtime block is one execution scope (a call frame or an if/for/range/switch/block scope), created per execution and pushed onto `Machine.Blocks`; distinct from a static block (`StaticBlock`, owned by a `BlockNode`, one per source node) and from file/package blocks (long-lived, referenced by `FuncValue.Parent` and persisted). Runtime blocks die when popped because closures capture heap items, not blocks.
- borrow rule: the three rules in `PushFrameCall` (`gnovm/pkg/gnolang/machine.go`) deciding whose realm authority a non-crossing call runs under, first match wins: #1 an `/r/`-declared callable borrows to its declaring realm; #2 a `/p/`-declared one with a real foreign-stamped receiver borrows to the receiver's constructing realm; #3 a `/p/` closure borrows to the realm active when it was constructed. Stdlib-stamped receivers are excluded from #2 and keep the caller's authority, since stdlib owns no mutable state.
- bptree (B+32): the versioned, copy-on-write B+tree with 32-way fanout (`tm2/pkg/bptree`, store wrapper `tm2/pkg/store/bptree`) that replaced IAVL as gno.land's `mainKey` state store; its shallower shape means a point read or write touches far fewer nodes than IAVL's binary descent, and its commitment structure differs, so swapping backends moves every app hash.
- chain: the on-chain stdlib root package (`gnovm/stdlibs/chain`); holds `Emit`, `Coins`, `PackageAddress`, with realm context in `chain/runtime` and the spoofable stack-walkers quarantined in `chain/runtime/unsafe`. Replaces the former `std` gno package.
- cold object read: a load of a persisted realm object not resident in the VM object cache, charged the flat store-read cost (`ReadCostFlat`, 59k gas, plus 17 gas/byte; `tm2/pkg/store/types/gas.go`); after cache GC a validator pays this per object on a traversal, so a data structure's cold gas scales with how many objects an operation touches.
- copy-on-write (COW): a persistent-tree update that, instead of changing a node in place, writes a new node and rewrites the path up to the root, leaving older versions untouched; how IAVL and the bptree share unchanged subtrees across versions.
- crossing / `cross`: a call into a crossing function (`func F(cur realm, ...)`), invoked as `cross(cur)`; the callee identifies its caller through `cur.Previous()` (see realm), not the stack-walking `unsafe.PreviousRealm()`.
- degraded mode: a gnoweb handler pattern where a failed secondary fetch is swallowed so the page still renders without that part (the overview substitutes an empty `JSONDocumentation` when qdoc fails); only the primary fetch, `ListFiles`, is allowed to fail the request.
- defined type: a type introduced by `type N U` (a `DeclaredType` in the GnoVM), distinct from its underlying type `U` and from an alias `type N = U`, which introduces no new type at all. A defined type carries its own method set, so `type N *T` has an empty one: Go forbids declaring methods on it and promotes none from `T`.
- depth-metered store: a tm2 store whose backend implements `types.DepthEstimator` (iavl, bptree), so its cache wrap charges tree-depth-scaled I/O gas (`DepthSet`/`DepthReadFlat`, `tm2/pkg/store/cache/store.go`) instead of the flat `WriteCostFlat`/`ReadCostFlat` a flat store like `dbadapter` charges. Moving a key between a depth-metered and a flat store changes its charged gas even when the bytes are identical.
- ephemeral realm: a short-lived code realm a `maketx run` script executes under (`gno.land/e/<addr>/run`); `realm.IsUser()` accepts it alongside true EOA calls while `realm.IsUserCall()` accepts only a direct EOA call, so a payment or auth guard that wants a real user must use `IsUserCall()`, not `IsUser()`.
- example test: a Go-style `ExampleXxx()` function in a `_test.gno` file whose trailing `// Output:` comment is compared against what the function prints; the `// Output:` line must begin the function's final comment group, or the runner treats the example as having no output and skips running it.
- Exception: the GnoVM's Go-level panic value (`gnovm/pkg/gnolang`), wrapping the gno panic value in `Exception.Value`; `runOnce` (`machine.go`) catches `*Exception` via Go `recover()` and re-raises anything else, so a bare Go panic escapes the VM and gno `recover()` (which returns `Exception.Value`) can't catch it.
- fast index: the bptree's optional key-to-value side map (`bptree.FastIndexOption`, mounted via `storebptree.FastStoreConstructor`) that serves a present-key point read on a clean working tree in one flat DB read instead of a tree descent; unauthenticated and outside the Merkle commitment, so it never moves the app hash or charged gas, paid for with one extra write per mutation and a duplicate of every live value on disk.
- filetest: a `*_filetest.gno` file executed by the VM and asserted against golden directives (`// Output:`, `// Error:`, `// Realm:`, `// Events:`, ...).
- flatten (interface): replacing an embedded-interface entry in an `InterfaceType.Methods` list with that interface's own methods (`flattenInterfaceMethods`), so a type's identity is its resulting method set rather than the name it embedded.
- function-local type: a `DeclaredType` declared inside a function body (`type S ...`), whose TypeID carries the declaration `Location` (`pkg[loc].Name`, `DeclaredType.IsFuncLocal`); unlike a package-level type it is not `SetType`'d at addpkg, so a persisted value referencing one needs the type written on the save path or it dangles on reload.
- gas: metered cost of CPU and memory during execution; consensus-relevant, so any change to it is a behavior change.
- genesis tx: a transaction embedded in `GenesisDoc.AppState.Txs` and executed by `InitChainer` at height 0, before any block. Fees are charged normally, so every signer needs a genesis balance; signature and sequence checks are skipped under `--skip-genesis-sig-verification`, which is what lets a deployment script patch a `MsgCall`'s `caller` field to an address whose key it does not hold.
- gas-wanted: the per-transaction gas budget the sender commits to (`tx.Fee.GasWanted`), installed as the meter's limit by `SetGasMeter` in the ante handler; exceeding it aborts the transaction while the fee is still charged, and simulation is metered against it too, so a dry run cannot discover a cost above the budget it was given.
- GnoConnect: the wallet-integration standard gnoweb implements (`docs/resources/gnoconnect.md`); a provider page advertises the chain through `gnoconnect:*` meta tags (`rpc`, `chainid`, `txdomains`) and expresses a transaction intent as a TxLink, so a client wallet can build the `MsgCall` without its own chain config.
- GnoVM: the Gno virtual machine (`gnovm/pkg/gnolang`) that preprocesses and interprets gno code.
- gnobuiltins: synthetic packages (e.g. `gnobuiltins/gno0p9`) injected only for type-checking, never run on chain.
- gnomod.toml: per-package manifest declaring the module path (pkgpath) and gno language version.
- gnoweb: the web frontend serving chain content (`gno.land/pkg/gnoweb`); server-rendered Go `html/template` views (realm, source, overview, help, user) fed by ABCI queries, with per-view Stimulus controllers compiled from TypeScript into `public/js/`.
- `gno test -p`: parallel mode of `gno test` (`gnovm/cmd/gno/test.go`); `-p N` tests N packages concurrently (`N <= 0` means GOMAXPROCS), each worker with its own Store, output buffered and printed in completion order. Surfaces global-state races that sequential `-p 1` hides.
- hard fork: a change to a persisted value's wire format or to a consensus gas constant; old state still decodes, but block bytes, IAVL hashes, or gas differ, so it ships only on fresh genesis or a coordinated upgrade.
- heap item: a heap-promoted variable slot (`HeapItemValue`), allocated for any local the preprocessor sees address-taken or closure-captured (`&local`, loop induction vars escaping into closures); a closure captures these, never the defining block, so the block can be freed once popped.
- lazy bind: an interface-formed method value (`g := i.M`, `defer i.M()`) held as a `BoundMethodValue` with `Func == nil`, whose concrete method and receiver are resolved at call time from the saved operand rather than at the bind; matches Go's call-time dispatch, so nil-panic timing, receiver snapshot and dynamic re-dispatch follow the operand's current value.
- namespace verifier: `gno.land/r/sys/names`, which gates `addpkg` on namespace ownership. Before its one-shot `Enable` runs every check passes; after it, a path `gno.land/{p,r}/<ns>/**` needs `<ns>` to be either the deployer's own address or a name the deployer currently holds in `r/sys/users`.
- new-real: an object marked reachable from the realm graph during a transaction but not yet assigned a persisted ObjectID (`ObjectInfo.GetIsNewReal`); the mid-transaction window before `assignNewObjectID` runs at finalize, distinct from a finalized (already-persisted) object.
- IAVL: the incumbent versioned, self-balancing Merkle binary tree backing tm2 state storage (`tm2/pkg/iavl`, store wrapper `tm2/pkg/store/iavl`); the bptree is a drop-in alternative.
- ICS23: the IBC vector-commitment proof standard (`cosmos/ics23/go`); a `ProofSpec` plus existence/non-existence `CommitmentProof`s let a light client verify membership/absence against a root. The bptree maps each node's mini-merkle to a uniform chain of binary `InnerOp`s.
- Machine: a GnoVM execution instance (`gno.Machine`) bound to a Store, Allocator, and Context.
- MemPackage: in-memory set of a package's source files (`std.MemPackage`, Go package `tm2/pkg/std`), the unit loaded, type-checked, and run.
- ObjectInfo: the per-object ownership and persistence header (`gnovm/pkg/gnolang/ownership.go`) holding the ObjectID, owner, refcount, and the dirty/escaped/deleted/new-real flags. Eight value types embed it directly (the `_ Object = &X{}` block), and the `BlockNode` family reaches it through `StaticBlock` → `Block`, so 20 concrete types satisfy `Object`. `*ObjectInfo` is the only type implementing the accessor set, so `oo.M()` and `oo.GetObjectInfo().M()` are interchangeable.
- persist-copy: the ref-collapsed copy of an object (`copyValueWithRefs`, `gnovm/pkg/gnolang/realm.go`) that amino-marshals to the store on `SetObject`, with child objects and declared types replaced by `RefValue`/`RefType` placeholders resolved on reload.
- preprocess: the static pass (`PredefineFileSet`/`initStaticBlocks`) that resolves names, types, and blocks before execution.
- private package: a package whose `gnomod.toml` sets `private = true` (`PackageValue.Private`); another realm may not persist a value whose type graph reaches into it, enforced at commit by `assertTypeIsPublic` (`gnovm/pkg/gnolang/realm.go`). The declaring realm is exempt from its own private types, and `maketx run` ephemeral packages are always private.
- privval: the validator's private-key signing subsystem (`tm2/pkg/bft/privval`), selecting one signer mode from config: a local `priv_validator_key.json` file, a gnokms remote signer the validator dials, or a tmkms/Horcrux listener the signer dials into. Every mode except tmkms is wrapped by the sign state gate.
- promoted field/method: a field or method of an embedded type made reachable on the embedding struct as if declared on it; Go's §Selectors rule resolves a name to the provider at the shallowest embedding depth, and ties at that depth (two providers, or one reached by two same-depth paths) make the selector ambiguous. `findEmbeddedFieldType` computes this in the GnoVM.
- pure package: an importable, stateless package under `p/`; contrast realm.
- qdoc: the `vm/qdoc` ABCI query returning a package's documentation as JSON (`doc.JSONDocumentation`: package doc, funcs, types, values, imports, and each symbol's declaring file and line); served with `unexported=true`, so callers that render only the exported API must filter it themselves.
- realm: a stateful on-chain package under `r/` whose objects persist across transactions; also the VM builtin type threaded as a `cur realm` parameter, where `cur.Previous()` returns the caller (unforgeable caller identity is `cur.Previous().Address()`, `cur.Previous().PkgPath()`), while bare `cur.Address()` / `cur.PkgPath()` are the current realm's own.
- RefNode: the lightweight stand-in (`gnovm/pkg/gnolang/nodes.go`, built by `toRefNode`) that a saved `FuncValue.Source` points to instead of the full source node; it carries only the node's `Location` (package path, file, and `Span` line/column numbers), so those line numbers are amino-serialized into every persisted realm object and read back, at gas cost, on each store load.
- RefType: the placeholder a persist-copy stores in place of a full type, a `RefType{ID: TypeID}` pointing at a `/t/<TypeID>` store record and resolved on reload via `GetType` (`fillTypesOfValue`); a `RefType` whose TypeID has no store record panics on load with "unexpected type with id ...".
- restricted denom: a coin denom listed in the bank `p:restricted_denoms` param that cannot be transferred unless the sending account is token-lock-whitelisted (`AccountUnrestricter.IsTokenLockWhitelisted`); enforced in `BankKeeper.canSendCoins` (`tm2/pkg/sdk/bank/keeper.go`). Distinct from vesting, which locks by schedule, not by denom.
- results hash: the per-block Merkle commitment to every tx's `ABCIResult` (`LastResultsHash` in the block header, built by `ABCIResults.Hash`, `tm2/pkg/bft/types/results.go`); each result contributes the amino-encoded bytes of its `Error`, `Data`, and `Events`, so any toolchain-dependent bytes in `ABCIResult.Error` diverge the hash and fork the chain.
- sealed interface: an interface carrying an unexported method, satisfiable only from that method's defining package; the unexported name is package-qualified, so a same-spelled method from another package is a distinct method rather than an implementation of it.
- selector: an `x.f` expression (`SelectorExpr`) naming a field or method, resolved by the preprocessor to a `ValuePath` after walking embedding for a promoted match; an unresolvable or ambiguous selector is rejected at preprocess and, for concrete types, already at type-check.
- session account: a delegated-signing account (`gnoland.GnoSessionAccount` wrapping `std.BaseSessionAccount`) created by `MsgCreateSession`, where a session key signs for a master address within an allow-path list, spend limit, and optional expiry; queried at `auth/accounts/{master}/session/{session}`.
- sign state: the validator's last-signed height/round/step record (`priv_validator_state.json`, `tm2/pkg/bft/privval/state`), consulted before every vote so the node refuses to sign twice at the same height and round. It is a local file, so it only serializes signing within one node; two nodes holding the same validator key each keep their own and will double-sign.
- signed zero: IEEE-754 keeps distinct `+0` and `-0`; Go constants do not, so any constant that converts to zero yields `+0` (golang/go#12621), while runtime float ops preserve the sign of an underflowed zero. `math.Signbit` distinguishes them.
- softfloat: the VM's deterministic software floating-point implementation (`gnovm/pkg/gnolang/internal/softfloat`) operating on raw IEEE-754 bit patterns (`uint32`/`uint64`); `F64to32` narrows float64 bits to float32, where `-0` is `1<<31`.
- spendable coins: an account's total balance minus its currently locked coins (`std.SpendableCoins`); for a vesting account the unlocked portion at a given block time, and the cap that `BankKeeper.SubtractCoins` enforces on every restricted transfer.
- stamp (origin package): recording a method's defining package in `FieldType.PkgPath` when that method is flattened out of another package's interface, so a cross-package unexported method stays distinct from a same-spelled local one; an empty stamp means the enclosing package, which `idName` falls back to.
- state sync: bootstrapping a node from a snapshot of committed state at a height (rather than replaying all blocks); the snapshot is streamed and rebuilt, then checked against the trusted app hash. The bptree's Exporter/Importer are its (not-yet-wired) state-sync surface.
- stdlib: gno standard libraries: prod under `gnovm/stdlibs/`, test-only overrides under `gnovm/tests/stdlibs/`.
- storage deposit: per-realm refundable charge for on-chain storage, locked on positive byte delta and refunded in proportion to the realm's original deposit ratio; `processStorageDeposit` (`gno.land/pkg/sdk/vm`), tracked as `rlm.Deposit`/`rlm.Storage`, governed by the `storage_price` and `default_deposit` VM params.
- Store: the backing store for packages and objects (`gno.Store`/`defaultStore`), layered over a tm2 CommitStore/IAVL.
- sub-realm token: a realm value minted by `cur.Sub(subpath)` (`gnovm/pkg/gnolang/uverse.go`) representing one of a realm's internal actors, with a synthesized `host#subpath` pkgpath and an address hashed from it; `#` is reserved in package paths, `Previous()` skips the host so chain depth is unchanged, and the token is ephemeral (never persisted). `chain.DerivePkgSubAddr` computes the same address off-chain; `chain.SplitPkgSubPath` and `realm.Subpath()` recover the host and subpath.
- transactionStore: the per-transaction Store wrapper (struct `{*defaultStore}`) returned by `BeginTransaction`, carrying per-tx caches and the tx-scoped gas meter; its methods promote to the embedded `*defaultStore`, so it is NOT matched by a `store.(*defaultStore)` type assertion.
- TxLink: the GnoConnect transaction-intent link a provider page emits (`docs/resources/gnoconnect.md`), naming the realm, the function, and named `&param=value` arguments; a wallet fills the rest from `vm/qdoc`, so a link may prefill only some arguments.
- txtar: testscript-based integration tests under `gno.land/pkg/integration/testdata/`.
- type-check: go/types-based validation of gno source (`TypeCheckMemPackage`), distinct from preprocessing.
- TypeCheckCache: per-run map of already-type-checked imported packages (`gno.TypeCheckCache`, `gotypecheck.go`), passed via `TypeCheckOptions.Cache` to skip re-checking; an unlocked map, so parallel `gno test` workers each hold their own.
- TypeID: a gno type's canonical string identity (`Type.TypeID()`, built in `types.go`) that decides type equality and is persisted in on-chain object state; structurally distinct types must get distinct TypeIDs, and changing the TypeID of an already-persisted type is consensus-breaking. For a struct/interface the id folds in each field/method entry's name, so an embedded field's derived name participates.
- unsafe: package `chain/runtime/unsafe` holding the stack-walking tx-origin primitives (`PreviousRealm`, `CurrentRealm`, `OriginCaller`, `OriginSend`); footgun-prone for auth (prefer `cur.Previous()`; see the catalog's Caller & access control).
- uverse: the GnoVM's universe block, the outermost scope holding built-in types, values, and functions (`gnovm/pkg/gnolang/uverse.go`, `makeUverseNode`); built-in-only names use the `uversePkgPath` and a leading `.` (e.g. `.uverse`, `.runtimeError`) so user code cannot spell them.
- ValuePath: the resolved access step (`ValuePath`, `types.go`) a selector compiles to, carrying a kind (`VPField`/`VPValMethod`/`VPPtrMethod`/`VPInterface`/deref variants), an embedding `Depth`, and an `Index`; a promoted match is a trail of these the VM follows at runtime via `GetPointerTo`.
- valoper: a validator's operator-keyed profile in `gno.land/r/gnops/valopers`, holding moniker, description and consensus pubkey. It is the management plane a validator uses to rotate its signing key or opt out, and the set `r/sys/validators/v3` proposals draw from; the operator address is deliberately distinct from the consensus signing address.
- vesting account: an `std.Account` (`tm2/pkg/std/vesting_account.go`) whose coins unlock on a schedule: continuous (linear `StartTime`→`EndTime`) or delayed (cliff at `EndTime`). Created only at genesis from a `Balance.Vesting` schedule; the bank keeper blocks transfers of still-locked coins and upgrades the account to a plain `BaseAccount` once fully vested.
